# 02 - Estruturador de Análise

## Quando usar
Antes de começar qualquer projeto de análise ou dashboard. Use para estruturar o raciocínio: o que precisa ser respondido, quais dados buscar, como limpar e como apresentar.

## Prompt

```
Atue como um Lead Data Analyst com experiência em projetos de BI e engenharia de dados.

Contexto:
- Ferramentas disponíveis: SQL Server, Power BI, Excel, Python
- Meu nível: analista em formação, em transição de carreira
- Domínio: [descreva o domínio — ex: controle de estoque, análise de vendas, painel fiscal]

Problema de negócio que preciso resolver:
[descreva em 2-3 frases o que o gestor ou time precisa saber]

Com base nisso, estruture um plano de análise completo com as seguintes etapas:

1. ENTENDIMENTO: Reformule o problema de negócio em uma pergunta analítica precisa
2. FONTES DE DADOS: Quais tabelas/colunas/sistemas provavelmente contêm os dados necessários
3. LIMPEZA: Quais problemas de qualidade de dados devo antecipar e como tratá-los
4. ANÁLISE: Quais métricas calcular, quais agrupamentos fazer, quais comparações são relevantes
5. VISUALIZAÇÃO: Como apresentar os resultados no Power BI (quais visuais, estrutura do dashboard)
6. COMUNICAÇÃO: Como apresentar os insights para quem não tem perfil técnico

Seja específico e prático. Priorize clareza sobre sofisticação.
```

## Notas de uso
- Preencha os campos entre [ ] com seu contexto real antes de enviar
- Use na fase de planejamento — antes de abrir o SQL ou o Power BI
